2.3 GENERACIÓN DE DATOS DE PRUEBA
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Implementación exitosa:
  ✓ Generador dinámico de emails con timestamp + UUID
  ✓ Patrón: testuser<timestamp><random>@test.com
  ✓ Garantiza: Unicidad en cada ejecución
  ✓ Beneficio: Eliminó necesidad de reset entre tests

Patrón reutilizable:
  ```
  function getRandomEmail() {
    const timestamp = new Date().getTime();
    const random = Math.floor(Math.random() * 100000);
    return 'testuser' + timestamp + random + '@test.com';
  }
  ```

Ventajas:
  • Tests completamente independientes
  • No requiere limpieza de datos
  • Escalable para múltiples ejecuciones paralelas

2.5 FLUJO END-TO-END
~~~~~~~~~~~~~~~~~~

Hallazgo de integración:
  ✅ Flujo Signup → Login funcionando perfectamente
  ✅ Credenciales dinámicas reutilizadas sin problemas
  ✅ Demuestra integración real entre endpoints
  ✅ Valida que datos creados son accesibles inmediatamente

Especificación del flujo:
  1. Generate random email: testuser<timestamp><random>@test.com
  2. Generate random password: ValidPass123
  3. Signup POST /signup { username, password }
  4. Validate: HTTP 200, Success message
  5. Login POST /login { username, password } (SAME credentials)
  6. Validate: HTTP 200, "Login successful."

Resultado: ✅ E2E test (CASO 3.4) PASS


2.6 RUNNERS Y EJECUCIÓN
~~~~~~~~~~~~~~~~~~~~~~

Evolución beneficiosa:
  Fase 1: SignupRunner (solo signup) - ❌ Incompleto
  Fase 2: LoginRunner (solo login) - ❌ Dos comandos separados
  Fase 3: AllTestsRunner (ambos) - ✅ Óptimo

Beneficio de AllTestsRunner:
  • Un comando para ejecutar 8 tests
  • ~11 segundos total (vs 19.8s con dos runners)
  • Ideal para CI/CD
  • Mantiene runners individuales para debugging

Comando recomendado:
  $ mvn clean test -Dtest=AllTestsRunner

4. Documentation First > Retroactive docs
